Этот файл содержит список ключевых рабочих процессов, команд и агентов, доступных в данном репозитории. Структура разделов должна быть минимально необходима для обеспечения читаемости и поддержки автоматической генерации документации.

## 🛠️ Основные Возможности (Core Capabilities)

Проект включает следующие функциональные блоки:

1. **Управление Файловыми Системами (Rclone)**:
   - Библиотека `rcloneTools.js` и директория `rclone/` обеспечивают взаимодействие с облачными хранилищами через командную строку `rclone`.
   - **Функционал**: Предоставляют набор функций для полного управления файловым контентом, включая синхронизацию, копирование, листинг, удаление (`rclone deletefile`) и, что критично, сканирование метаданных (`rclone lsjson`).
- **Ключевые рабочие процессы**:
  - **Синхронизация/Копирование**: Функционал основан на прямых вызовах `rclone` CLI для обеспечения соответствия локального и удаленного каталогов (sync, copy, ls).
  - **Управление Дубликатами (LLM-Guided Deduplication)**: Использование `rcloneTools.getOnlyDuplicateGroups` для надежного сканирования всех файлов через MD5 хэши, автоматической группировки дубликатов и их последующей отправки в LLM для принятия решения.
- **Управление Структурой и Оптимизацией**: Новая функциональность, объединяющая сбор полного каталога с расчетом метрик (количество файлов, общий размер), и предложением оптимизационных стратегий для улучшения структуры данных.

2. **Управление Промптами и Состоянием (Agent Prompt Management)**:
   - Файл `agentPrompt.js` используется для генерации структурированных, машиночитаемых данных (payload) и мастере-промпта для направляющего ИИ-агента.
   - **Ключевые рабочие процессы**:
     - **Управление дедупликацией (LLM-Guided Deduplication)**: Выполняется в несколько этапов. Сначала собирается сырые данные о дубликатах (хэш, список файлов) с помощью `rcloneTools` (Шаг 1: Discovery). Затем эти данные структурируются в JSON-payload (`buildPromptPayload`) и передаются в LLM. Агент принимает решение, какой файл является "лучшей" копией, и возвращает список относительных путей к файлам, которые необходимо удалить, что минимизирует риск ошибки LLM.

3. **Центральный Контроллер (Main Logic) и Оркестрация**:
   - `index.js` является ядром системы и главным оркестратором. Он координирует весь многошаговый рабочий процесс.
   - **Ключевые рабочие процессы (Orchestration Flow)**:
     1. **Инициализация**: Инициализирует компоненты (`rcloneTools`, `agentPrompt`).
     2. **Обнаружение (Discovery)**: Идентифицирует ресурсы, требующие обработки (например, дубликаты файлов с помощью `rcloneTools.getOnlyDuplicateGroups`).
     3. **Принятие решения (Decision Making)**: Формирует структурированный промпт/payload и передает его в LLM через `agentPrompt.js`. LLM принимает критически важное решение, которое затем используется для формирования списка действий.
     4. **Исполнение (Execution)**: Выполняет финальные действия, такие как удаление файлов (`rcloneTools.deleteFile`) или синхронизация (`rclone sync`), основываясь на решениях, принятых LLM.

## 🚀 Команды (Commands)

Ниже перечислены основные CLI команды или вызовы API, которые реализуются через этот набор инструментов.

| Команда/Вызов | Описание | Используемые компоненты |
| :--- | :--- | :--- |
